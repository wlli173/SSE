Arquitetura

Esta aplicação é dividida em dois containers Docker: o sse-server (Node.js + Express) gera as cotações e expõe o stream /events; o sse-client (nginx) serve esta SPA e faz proxy reverso para o backend. O navegador sempre fala com a mesma origem — não há CORS, e o proxy preserva o stream com proxy_buffering off.

No código JavaScript: new EventSource('/events'). O servidor mantém a conexão HTTP aberta e empurra mensagens no formato text/event-stream sempre que houver novidades — sem polling, sem WebSocket.
